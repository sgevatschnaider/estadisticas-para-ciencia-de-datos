Proceso Gaussiano + Programación Genética

Esta simulación representa una idea central de investigación: usar programación genética para evolucionar kernels válidos de un proceso gaussiano. En lugar de fijar de antemano un kernel RBF, periódico o racional cuadrático, la programación genética construye expresiones simbólicas a partir de componentes base y operaciones de suma o producto.

La restricción importante es matemática: la demo solo combina kernels que preservan semidefinitud positiva. Por eso se usan bloques como RBF, lineal, periódico, racional cuadrático y constante, junto con suma y producto. Así, el proceso gaussiano sigue siendo un modelo probabilístico coherente sobre funciones.

Qué hace el modelo

Primero se generan datos sintéticos de entrenamiento. Luego una población de árboles-kernel compite según su capacidad predictiva. Cada individuo define una geometría distinta de semejanza entre entradas, y esa geometría induce una media posterior, bandas de incertidumbre y una matriz de covarianza diferente. El objetivo evolutivo de esta demo es reducir el error predictivo sobre una grilla densa de validación y, al mismo tiempo, mostrar el log marginal likelihood como diagnóstico probabilístico complementario.

Aplicaciones de esta metodología

Una aplicación natural es la descubierta automática de kernels: en series temporales, señales científicas, datos espaciales o mediciones periódicas, la programación genética puede proponer estructuras que mezclen tendencia, suavidad, repetición y escalas múltiples. También es útil en modelado simbólico asistido por probabilidad, porque combina interpretabilidad estructural con cuantificación de incertidumbre.

Otra aplicación aparece en optimización evolutiva asistida por modelos sustitutos. Allí la programación genética puede buscar programas, reglas o kernels, mientras que el proceso gaussiano resume lo aprendido y ayuda a comparar candidatos con menos evaluaciones costosas. En ingeniería, control, química computacional, diseño de materiales y calibración de simuladores, esta lógica resulta especialmente valiosa cuando cada evaluación real es cara.

Desde un punto de vista conceptual, esta demo une tres ideas: una representación simbólica del kernel, una geometría probabilística definida por la covarianza y una búsqueda evolutiva que explora el espacio de estructuras posibles. No evoluciona programas arbitrarios, sino funciones de covarianza que cambian la forma en que el proceso gaussiano entiende el espacio de entrada.
